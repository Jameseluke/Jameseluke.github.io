Responsiveness and JS

James Luke
2017-04-06
Design, University, Web

Website is now hosted online

This week we were challenged with making our site responsive and less static, in order to make for a more enjoyable user experience across all platforms.

Luckily due to the inclusion of the bootstrap framework within our project from the very start, the website was fairly responsive to begin with.

Some of the key elements that needed to be responsive were the navbar, which was achieved by collapsing the right aligned navlinks when the screen was reduced in size, the image galleries, as the images needed to scale to fit the screen rather than only have half an image viewable at a time, and the spaces page, as the grid format would cause each space to be too small on mobile devices.

The initial solution for the spaces page, was to put each set of three spaces in a row. This created a nice layout at large resolutions, but when the screen size was reduced, it began to break down. Grid elements would either become too small, or would wrap onto the next line into a 2,1,2,1,2,1 column format. However, this was fixed by including all of the elements in one row, with the proper col-*-* classes set.

Keeping the images responsive was mostly handled through the img-fluid class built into bootstrap, however some minor tweaking was needed in order for our needs to be fully met. The img-fluid class cannot be inside a flex-box div, something which caused a few issues before this was understood. Furthermore, if min-width was not set for the image wrapper, then the image would stay small, regardless of the size of the surrounding div.

Navbar collapse was handled mostly with bootstrap classes, however in order to allow the navbar to stick to the top of the webpage after it has been scrolled past on the landing page (position: fixed, top 0), a few additional styles needed to be applied to it. This included “min-width: 100%” as the navbar was losing its flex layout when set to fixed position.
