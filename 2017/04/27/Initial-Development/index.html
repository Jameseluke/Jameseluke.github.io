Initial Development

James Luke
2017-04-27

This week our goal was to create a vertical slice prototype of the system. The functionality we chose to focus on was the creating canvases page. The purpose of this page is to allow users (who have been logged in) to create a canvas, complete with a title, details about the space they are offering, and the address. This task is an ideal place to start as it is a core functionality of our product, and it incorporates the use of the whole MEAN stack.

The structure is rather simple. We have a create canvases page which is simply a form. The user enters in the details of the canvas they are creating in the necessary fields, and clicks submit. As of now, the only fields we have is the title of the canvas, the description, and the address, each represented as a String. We’ve set up an angular controller for creating canvases, with a submit function that binds the data collected from the form to the scope, and then sends it to the server via a post request. Our server side uses express js, and we use the express Router to receive the json object, create a new Canvas object and save this to the Canvas model.

If it was successfully saved to the mongoose database system, then we should be able to see the json object in the canvases page. A similar streamline process is used to allow us to see all the json objects on the canvases page. Using Angular we had to make a get request to receive all the canvas objects in the database. This request is sent to the server side, and we query the database accordingly, sending the appropriate response back.

In addition to this prototype of the system, we also made the sign up and log in page functional.

These were challenging tasks since a lot of the technologies we’re using is rather new to us (AngularJS, ExpressJS, NodeJS, Mongoose, other middleware, ect), but it was really informative and we learnt a lot in the process.

Future Endeavours

This is a very basic prototype of what we hope will be more polished and elaborate. We’re hoping to improve the create canvas page in many different areas. For instance, we want to incorporate the user id as part of the Canvas object so we know who the owner is, which we will utilise to allow the owner to edit their own canvases (if they are logged in of course). We also want to have the capability to upload images as part of the canvas object, and this image will be displayed on the canvas page. We have researched this and plan to use multer library with ngFileUpload (and angular directive). Finally we also want to look into making the address of the canvas more definite. At the moment it is just a single String, but we are looking into the idea of making it several different components, and somehow determine the latitude and longitude coordinates of the given address which will be useful for our mapping functionality.

The next functionality we will look into from here is allowing artists to submit their ideas to a given canvas. We’re hoping this will be a slightly easier process as it will be similar to how we created a canvas.
